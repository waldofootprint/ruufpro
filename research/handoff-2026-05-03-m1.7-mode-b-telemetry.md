## What is Mode B?

Mode B = the calculator's refuse path. When measurement fails or the address looks wrong, the API returns `error_code: 'measurement_unavailable'` (or similar) and the V4 widget shows a "manual quote" message instead of a price.

Triggers per [`app/api/estimate/route.ts`](app/api/estimate/route.ts) and [`lib/measurement-pipeline.ts`](lib/measurement-pipeline.ts):

- Geocoding non-residential → Mode B
- Geocoding unrecognized → Mode B
- LiDAR + Solar both fail + footprint area exists → Mode B
- Footprint also missing → Mode B
- Sanity guardrail (sqft <600 or >10K or neighbor-grab) → Mode B

**We currently log NONE of these.** Zero telemetry. We don't know baseline rate.

### 2. Log Mode B fires

In [`app/api/estimate/route.ts`](app/api/estimate/route.ts), find every place that returns an error response with an error_code (search for `error_code:` in the file). Before each `return NextResponse.json({ error..., error_code: ... }, { status: 400 })`, fire:

```ts
await supabase.from("widget_events").insert({
  contractor_id,
  event_type: "mode_b_fired",  // or whatever the existing column is called
  metadata: {
    error_code,
    address: address || null,
    pipeline: pipelineUsed || null,
  },
}).then(() => {}, () => {}); // fire-and-forget, never block the response
```

Make this a single helper at the top of the file:

```ts
async function logModeB(contractor_id: string, error_code: string, address: string | null, pipeline: string | null) {
  try {
    await supabase.from("widget_events").insert({
      contractor_id,
      event_type: "mode_b_fired",
      metadata: { error_code, address, pipeline },
    });
  } catch {
    // never block the response on telemetry failure
  }
}
```

Call it before every error-code return. Don't call on validation 400s (`contractor_id required`, etc.) — only on measurement/geocoding/guardrail failures.

### 3. Daily roll-up endpoint

New file `app/api/dashboard/insights/mode-b-rate/route.ts`:

```ts
import { NextRequest, NextResponse } from "next/server";
import { createServerSupabase } from "@/lib/supabase-server";

export async function GET(req: NextRequest) {
  const supabase = createServerSupabase();
  const { data: { user } } = await supabase.auth.getUser();
  if (!user) return NextResponse.json({ error: "Unauthorized" }, { status: 401 });

  // Resolve contractor_id from logged-in user
  const { data: contractor } = await supabase
    .from("contractors")
    .select("id")
    .eq("user_id", user.id)
    .single();
  if (!contractor) return NextResponse.json({ error: "No contractor" }, { status: 404 });

  // Count fires + total estimate attempts in last 7 days
  const sevenDaysAgo = new Date(Date.now() - 7 * 24 * 60 * 60 * 1000).toISOString();

  const [{ count: fires }, { count: totalAttempts }] = await Promise.all([
    supabase
      .from("widget_events")
      .select("*", { count: "exact", head: true })
      .eq("contractor_id", contractor.id)
      .eq("event_type", "mode_b_fired")
      .gte("created_at", sevenDaysAgo),
    supabase
      .from("widget_events")
      .select("*", { count: "exact", head: true })
      .eq("contractor_id", contractor.id)
      .in("event_type", ["mode_b_fired", "estimate_complete"])  // adjust to whatever the success event is
      .gte("created_at", sevenDaysAgo),
  ]);

  const ratePct = totalAttempts && totalAttempts > 0
    ? Math.round((fires! / totalAttempts) * 1000) / 10
    : 0;

  return NextResponse.json({
    fires_7d: fires || 0,
    total_attempts_7d: totalAttempts || 0,
    rate_pct: ratePct,
  });
}
```

Confirm the success event name first by reading any existing `widget_events.insert` call in the codebase — there should already be one for completed estimates.
